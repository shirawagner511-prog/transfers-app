# מערכת ניהול העברות פנימיות

מערכת ניהול לעסקי מזון: העברות פנימיות, מרכיבים, מוצרים, ספקים, ומחלקות.

## הגדרה ראשונית

### 1. הכנת Supabase

1. פתח חשבון ב-https://supabase.com וצור פרויקט חדש.
2. בלוח הבקרה: **SQL Editor** → הדבק ורץ את `supabase/migration.sql`
3. יצור משתמש ראשון: **Authentication → Users → Add user**
4. עדכן הרשאה ל-admin:
   ```sql
   update user_profiles set role = 'admin' where user_id = '<user-id>';
   ```

### 2. משתני סביבה

```bash
cp .env.example .env
```

מלא ב-.env:
- `VITE_SUPABASE_URL` — Project Settings > API > Project URL
- `VITE_SUPABASE_ANON_KEY` — Project Settings > API > anon public

### 3. הרצה

```bash
npm install
npm run dev
```

## הרשאות

| role    | צפייה | עריכה | אישור | ניהול משתמשים |
|---------|-------|-------|-------|----------------|
| admin   | ✓     | ✓     | ✓     | ✓              |
| manager | ✓     | ✓     | ✓     | ✗              |
| editor  | ✓     | ✓     | ✗     | ✗              |
| viewer  | ✓     | ✗     | ✗     | ✗              |

## רשימת בדיקות ידניות

1. כניסה למערכת עם מייל וסיסמה
2. הוספת מחלקה חדשה (מחלקות)
3. הוספת ספק (ספקים)
4. הוספת מרכיב עם מחיר (מרכיבים)
5. עדכון מחיר מרכיב ← בדוק שנרשמה היסטוריה
6. הוספת מוצר עם מרכיבים ← בדוק חישוב עלות
7. יצירת העברה פנימית ← שליחה לאישור
8. אישור העברה כ-manager
9. פתיחת סיכום יומי מלוח הבקרה
10. לחיצה על העברה בציר הזמן
11. שינוי מחיר מרכיב ← בדוק שהעברות ישנות לא השתנו
12. בדיקת RTL ומובייל
